Getting Started

API Garage is a two part software: 1) Backend for persisting the information and 2) a cross platform frontend application.

Setting up the backend

Setting Up LAMP Stack

Easiest way to setup a LAMP stack is to use Tasksel. Tasksel is a LAMP stack utility to install PHP, MySQL, and Apache. Once we have a docker container for API Garage.

$ sudo apt-get install tasksel
$ sudo tasksel # Select MySQL, PHP, and Apache for installation.

Login to MySQL.

$ mysql -u root -p # Use the password created while setting up Tasksel

Setup a database and a user.

# Create a database
CREATE DATABASE ;

# Create a new user
CREATE USER ''@'localhost'
IDENTIFIED BY '';
# Grant privileges to the user
GRANT ALL PRIVILEGES ON *.* TO ''@'localhost'
WITH GRANT OPTION;

Setup The Repository

Backend is written in PHP Larave 4. The repository is located at https://github.com/apigarage/ag-backend. Clone the repository on the server.

$ git clone git@github.com:apigarage/ag-backend.git

Laravel is using a PHP dependency manager. If you don't have it installed, use Composer's Getting Started to install it. Once the Compser is installed, install the code dependencies. This will take some time.

$ composer install
OR
$ /usr/local/bin/composer.phar install

Next is to setup the database configuration for the application. Create a new file on the root directory named .env.local.php . Below is an example of what it will look like.


return array(
/* Database */
'db_type' => '_______',
'db_host' => '_______',
'db_name' => '_______',
'db_username' => '_______',
'db_password' => '_______',

API Garage sends an email when user adds a comment to an endpoint. It's using Mailgun to send emails. You can sign up for free mailgunn acountnt aand generate the keys for setup. Add the config below to the .env.local.php file.

...

/* mailgun */
'mailgun_domain' => '_________',
'mailgun_secret' => 'key-_________',
);


Setting up Fronend

API Garage is a desktop application created in Electron. It's purpose is to generate live documentation for teams to collaborate on during the development process.

Two main requirements for the projects are node and bower. You can install node from node website. You can install bower from bower website.

Setup the code

Clone the repository.

$ git@github.com:apigarage/ag-frontend.git

Install node and bower dependencies.

$ cd ag-frontend
$ npm install
$ cd browser/app
$ bower install

Inside the config folder create a config file based on the that will be building against filename has to be called env_.json.

Create a env_development.json with the contents below.

{
"name": "development",
"description": "Add here any environment specific stuff you like.",
"url": "<>",
"api": "api/",
"client_id": "id_<>",
"client_secret": "secret_<>",
"database_name": "staging-apigarage",
"access_token_key": "accessToken_staging",
"manifest_file_url": "",
"mixpanel_key": "",
"update_check_interval": 60000,
"syncTime": 10000
}

The project is ready to go. Run the command below to start application

npm start

If you want to build an installer from the frontend client, details can be found at the proposed deployment flow . Currently the deployment script supports all three major operating systems linux, windows, and osx.
